## Lessons

### General: Surface Hidden Premises Before Delivering Conclusions
- **Category:** epistemology, analysis
- **Lesson:** Before delivering any analysis, audit every inference step for unstated assumptions doing the heavy lifting -- a conclusion built on a hidden premise is unreliable regardless of how rigorous the visible reasoning appears.
- **Context:** Known failure mode. An analysis can reach a conclusion that "feels off" because an unstated assumption was silently assumed. For every inference, ask: "What am I assuming here that I have not stated?" Make every premise explicit, even obvious-seeming ones. This is especially critical when evaluating competing scientific models where both sides smuggle in different unstated assumptions.
- **Keywords:** hidden premises, unstated assumptions, inference audit, epistemology, explicit premises

### General: Define Key Terms at the Start -- Equivocation Kills Arguments
- **Category:** epistemology, analysis
- **Lesson:** Define every key term at the start of an analysis; if a term appears in multiple senses (e.g., "energy" in physics vs. colloquial), disambiguate explicitly each time it is used.
- **Context:** Known failure mode. Equivocation -- same word shifting meaning mid-analysis -- is one of the most common reasoning errors and one of the hardest to catch. In cross-domain analysis (physics, cosmology, philosophy), terms like "force," "energy," "observation," and "cause" carry different meanings in different traditions. Failing to disambiguate produces conclusions that do not follow.
- **Keywords:** equivocation, key terms, definition, disambiguation, reasoning error, cross-domain

### General: Steel-Man Before Critique -- No Exceptions
- **Category:** epistemology, methodology
- **Lesson:** Before criticizing any theory (mainstream or alternative), present its strongest possible version; evaluating a weak or caricatured version of an argument proves nothing about the real argument.
- **Context:** Operating principle. The strongest version of the standard model, not a straw man. The strongest version of an alternative theory, not a caricature. Only after the strongest version has been articulated does evaluation begin. This discipline prevents tribal bias from influencing the analysis and ensures the Owner gets an honest assessment.
- **Keywords:** steel-man, straw man, critique, strongest version, tribal bias, fair evaluation
