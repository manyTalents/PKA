## Lessons

### General: Assess Before Programming -- Never Skip the Baseline
- **Category:** performance, programming
- **Lesson:** Always assess current training history, fitness level, equipment access, injury history, and recovery capacity before prescribing any program -- a program without a baseline is a guess.
- **Context:** Known failure mode. A program that is too advanced risks injury; too easy and there is no stimulus. The first interaction is always assessment, not programming. This is especially critical for the Owner's profile: 50-60 hour work weeks with a chaotic schedule mean recovery capacity is lower than an athlete's.
- **Keywords:** assessment, baseline, training history, injury, recovery capacity, fitness level

### General: Design for the Real Schedule, Not the Ideal One
- **Category:** performance, lifestyle
- **Lesson:** Every program must be designed for the Owner's actual constraints (50-60 hour work week, chaotic schedule, home gym) -- and every protocol must have a full version AND a minimum viable chaos-mode version.
- **Context:** Known failure mode. One-size-fits-all programming assumes a perfect schedule, full gym access, and 8 hours of sleep. It breaks immediately in the real world. The system must degrade gracefully: from the full 5-day program down to the 3-hour minimum viable week (2 resistance + 2 Zone 2 + sleep). The system bends; it never breaks.
- **Keywords:** chaos mode, minimum viable, graceful degradation, real schedule, constraints, entrepreneur

### General: Adherence Is the Number One Training Variable
- **Category:** performance, programming
- **Lesson:** A good program executed consistently beats a perfect program executed sporadically -- ship a good-enough program and iterate, rather than over-optimizing before the Owner starts.
- **Context:** Known failure mode (over-optimization paralysis). Per the 2026 ACSM Position Stand, the largest effect size comes from going from no resistance training to any resistance training. Obsessing over the perfect periodization model or supplement timing instead of shipping an executable program is a net negative. Consistency is the primary variable; everything else is secondary.
- **Keywords:** adherence, consistency, over-optimization, ACSM, ship it, iterate, training variable
